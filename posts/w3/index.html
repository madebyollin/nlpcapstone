Project Proposal

We propose building a sentiment-aware text-to-speech system.

This means accomplishing the following three core objectives:

  • Collecting Sentiment-Tagged Data
  • Creating a Multi-Class Sentiment Classifier
  • Creating a Sentiment-Conditional Text-to-Speech System

Intended Methodologies

Collecting Sentiment-Tagged Data:

  • Twitter Sentiment Scraper: Pull a selection of English tweets from Twitter, normalize characters, filter for sentiment-corresponding emoji (e.g. anger: {ðŸ˜¡, ðŸ˜¤, ðŸ˜ , ðŸ‘Ž}) and tag appropriately (with the emoji removed).
  • Audio Sentiment Scraper: If we have a sufficiently good classifier, we can extract audio snippets from Films/TV shows using subtitles, tag them using our existing classifier, and then use the most data which the classifier is most certain about to train more advanced speech generation systems.

Creating a Multi-class Sentiment Classifier

  • Baseline Classifier: A simple unigram bag-of-words model.
  • LSTM Classifier: A Bidirectional LSTM with custom embeddings.
  • Better LSTM Classifier: Improvements to our classifier.

Creating a Sentiment-Conditional Text-to-Speech System

  • Hard-Coded Pitch Envelopes: Dead simple, hard-coded pitch envelope application system using aupyom and/or librosa, based on known features of our emotional classes.
  • Conditional TTS: Addition of local conditioning (on words) as well as sentiment conditioning to TF-WaveNet

Evaluation Plan

  • Sentiment Classifier Evaluation: We can batch our classes into positive/negative and test against a standard dataset (like the IMDB movie reviews dataset).
  • Full System Evaluation: We will set up a demo page (either running locally or on a webserver) for testing.
  • TTS Mean Opinion Score: We can compare the MOS between unconditional and sentiment-conditional renditions of text, surveyed from MTurk (or possibly from classmates).
